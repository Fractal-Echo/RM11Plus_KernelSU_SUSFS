## Android 16 / GKI 6.12 Rules

Primary references:

- Android 16 / 6.12 GKI release builds:
  <https://source.android.com/docs/core/architecture/kernel/gki-android16-6_12-release-builds>
- Stable KMI:
  <https://source.android.com/docs/core/architecture/kernel/stable-kmi>
- Loadable kernel modules:
  <https://source.android.com/docs/core/architecture/kernel/loadable-kernel-modules>
- Kernel modules overview:
  <https://source.android.com/docs/core/architecture/kernel/modules>

Practical rules for this project:

- KMI stability is scoped to the same Android release and LTS pair, here
  `android16-6.12`.
- Build kernel and modules with the toolchain tied to the target branch.
- Vendor modules must remain compatible with the KMI and exported symbol surface.
- GKI module signing is not a blanket permission model for arbitrary vendor
  modules. Unsigned modules may load only when they use allowed symbols or
  symbols provided by other unsigned modules.
- `CONFIG_MODULE_SIG_FORCE` must stay disabled for this target.
- `CONFIG_MODULE_SIG_ALL` should be disabled for NX809J device kernel builds so
  non-GKI modules are not automatically treated as GKI-signed artifacts.
- Disabling `CONFIG_MODULE_SIG_ALL` is not the same as bypassing AVB, vbmeta, or
  bootloader verification.

## Required NX809J Kernel Config

These options must be present in the final `.config`, not only appended to
`gki_defconfig`:

```text
CONFIG_KSU=y
CONFIG_KSU_SUSFS=y
CONFIG_NTSYNC=y
CONFIG_OVERLAY_FS=y
CONFIG_TMPFS_XATTR=y
CONFIG_TMPFS_POSIX_ACL=y
CONFIG_MODULE_SIG=y
# CONFIG_USER_NS is not set
# CONFIG_MODULE_SIG_FORCE is not set
# CONFIG_MODULE_SIG_ALL is not set
```

The current Droidspaces v6.3.0 binary reports all required features available
with `CONFIG_USER_NS` disabled and marks that disabled state as hardening-positive.
`CONFIG_CGROUP_PIDS`, `CONFIG_DEVTMPFS_MOUNT`, and `CONFIG_IP6_NF_NAT` are useful
to track in build logs, but they are warnings unless a tested container/rootfs
proves one of them is required.

## On-Device Validation Commands

Use `adb shell su -c ...` directly. Avoid wrapping the whole command in an extra
remote shell string unless quoting has been tested.

Kernel and KernelSU:

```powershell
adb shell su -c id
adb shell su -c uname -a
adb shell su -c /data/adb/ksud --version
adb shell su -c '/data/adb/ksud module list'
```

Kernel config:

```powershell
adb shell su -c 'zcat /proc/config.gz | grep -Ei "CONFIG_(KSU|KSU_SUSFS|NTSYNC|OVERLAY_FS|TMPFS_XATTR|TMPFS_POSIX_ACL|USER_NS|CGROUP_PIDS|DEVTMPFS_MOUNT|IP6_NF_NAT|MODULE_SIG|MODULE_SIG_FORCE|MODULE_SIG_ALL)"'
```

Droidspaces module:

```powershell
adb shell su -c 'ls -la /data/adb/modules/droidspaces'
adb shell su -c 'ls -la /data/local/Droidspaces /data/local/Droidspaces/bin'
adb shell su -c '/data/local/Droidspaces/bin/droidspaces check'
adb shell su -c 'ps -A -o USER,PID,PPID,NAME,ARGS | grep -i droidspaces'
adb shell su -c 'cat /data/local/Droidspaces/Logs/module-setup.log'
```

Module signature state:

```powershell
adb shell su -c 'cat /sys/module/module/parameters/sig_enforce 2>/dev/null || true'
adb shell su -c 'cat /proc/sys/kernel/modules_disabled 2>/dev/null || true'
adb shell su -c 'lsmod | head -40'
```

## Build and Flash Safety

Build target:

- Use `.github/workflows/build_nx809j.yml`.
- The workflow target model is `OPNX809J`, not `OP15`.
- The workflow must fail if the required config values are missing after
  `olddefconfig`.

Safe install order:

1. Keep a known-good boot backup.
2. Prefer `fastboot boot` for a new test image when a bootable image is
   available.
3. Flash permanently only after boot, root, module list, and Droidspaces checks
   pass.
4. Reboot after installing or enabling the Droidspaces KernelSU module.
5. Check `/data/local/Droidspaces/Logs/module-setup.log` if the daemon payload is
   missing after reboot.
6. If the payload exists but the daemon is not running, verify `.daemon_mode` is
   `1` and inspect `/data/local/Droidspaces/Logs/boot-module.log`.

## What Can and Cannot Be Fixed Here

Can be fixed in this repo:

- Kernel config fragments and workflow validation.
- AnyKernel packaging metadata and executable bits.
- Droidspaces KernelSU module ZIP packaging, payload copying, permissions, and
  logs.
- Documentation and reproducible validation commands.

Cannot be guaranteed only by this repo:

- Loading arbitrary third-party `.ko` files that use non-KMI or unavailable
  symbols.
- Bypassing AVB/vbmeta/bootloader policy.
- Making an incompatible vendor module compatible without rebuilding it against
  the correct kernel/KMI.
- Proving Droidspaces userspace works without the Droidspaces APK/rootfs and a
  rebooted, active module.
